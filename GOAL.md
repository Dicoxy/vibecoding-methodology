# GOAL — Методология блочного кодинга

**Цель:** Создать повторяемый, документированный процесс разработки ПО с AI, который позволяет маленькой команде (2-3 человека) выпускать продукты на уровне, обычно требующем 10-20 человек и 6-12 месяцев.

---

## Измеримые критерии

### Критерий 1: Скорость запуска нового проекта
- **Сейчас:** каждый проект запускается вручную, настройка окружения и контекста занимает дни
- **Цель:** новый проект стартует за 2-4 часа (GOAL → BLOCKS → contracts → первый блок в работе)
- **Как проверить:** запустить Tier 2 проект по методологии, замерить время от идеи до первого рабочего блока

### Критерий 2: Передаваемость
- **Сейчас:** только Гриша может запустить и вести проект
- **Цель:** человек с техническим бэкграундом может вести проект по методологии после 1-2 часов онбординга
- **Как проверить:** дать Маше или новому человеку запустить проект по шаблонам

### Критерий 3: Сохранение контекста
- **Сейчас:** контекст теряется между сессиями, AI не знает что было решено
- **Цель:** любая AI-сессия получает полный контекст проекта за <10K токенов через RAG
- **Как проверить:** начать сессию с чистого контекста, задать вопрос про прошлое решение — RAG находит ответ

### Критерий 4: Качество на выходе
- **Сейчас:** MARS за 5 недель (2 человека) — доказательство работоспособности
- **Цель:** повторить результат на 2-3 разных проектах (CRM, KIBA, sovr-resheniya)
- **Как проверить:** каждый проект укладывается в оценку ±30%

## Не-цели (что мы НЕ делаем)

- Не создаём фреймворк или библиотеку (это документ + процесс)
- Не автоматизируем полностью (человек-архитектор ВСЕГДА в цикле)
- Не привязываемся к конкретным инструментам (Claude, Cline, Cursor — заменяемы)
- Не пишем академическую работу (практика важнее теории)
